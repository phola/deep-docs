# Phase 9: Package

Generate a hostable documentation site from the L1-L4 output using MkDocs Material.

## Why MkDocs Material

- Markdown-native — L1/L2/L3/L4 files drop in without rewriting
- Built-in Mermaid diagram rendering
- Full-text search across all docs
- Responsive navigation sidebar with collapsible sections
- Dark mode, syntax highlighting, admonitions
- Static site output — deploy anywhere (Azure Static Web Apps, GitHub Pages, Netlify, S3, or serve locally)
- Lightweight: `pip install mkdocs-material` + `mkdocs build`

## Prerequisites

The orchestrator checks for `mkdocs` and `pip` availability. If missing, it logs
instructions and skips the build step (the generated `mkdocs.yml` and site structure
are still useful for manual setup).

## Sub-agent Task

Spawn one sub-agent to generate the site configuration and structure:

```

Generate these files:

1. {{DOCS_DIR}}/mkdocs.yml — MkDocs configuration:

        icon: material/brightness-4

## Build Step

After the sub-agent completes, the orchestrator runs:

```bash
# Check if mkdocs is available
if command -v mkdocs &>/dev/null; then
  cd {{DOCS_DIR}}
  mkdocs build --strict 2>&1
  if [[ $? -eq 0 ]]; then
    log "PACKAGE — site built to {{DOCS_DIR}}/site/"
  else
    log "PACKAGE — ⚠️ mkdocs build had warnings (site may still be usable)"
  fi
else
  log "PACKAGE — mkdocs not installed. To build: pip install mkdocs-material && cd {{DOCS_DIR}} && mkdocs build"
fi
```

## Local Preview

```bash
cd {{DOCS_DIR}} && mkdocs serve
# Opens at http://localhost:8000
```

## Deployment Options

The generated `site/` directory is a static site. Deploy to:

| Platform | Command |
|----------|---------|
| GitHub Pages | `mkdocs gh-deploy` |
| Azure Static Web Apps | Upload `site/` or use GitHub Actions |
| Netlify | Point build to `mkdocs build`, publish `site/` |
| S3 / CloudFront | `aws s3 sync site/ s3://bucket-name` |
| Local / Docker | `python -m http.server -d site/` or nginx |
